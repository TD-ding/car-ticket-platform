# 管理后台功能说明

## 访问权限

仅 `is_admin=True` 的用户可访问，非管理员访问返回 403。管理员可通过以下方式创建：

- 首次运行时自动创建 `admin / admin123`
- 已有管理员在用户管理页切换其他用户的管理员身份

## 导航

管理后台使用独立的侧边导航 (`admin/_nav.html`)，包含四个入口：

- 仪表盘
- 班次管理
- 订单管理
- 用户管理

## 仪表盘 (/admin)

展示四个统计卡片：

| 指标 | 数据来源 |
|------|----------|
| 注册用户 | `User.query.count()` |
| 活跃班次 | `Schedule` status=active |
| 有效订单 | `Order` status=paid |
| 累计营收 | `SUM(Order.price)` status=paid |
| 候补中 | `Waitlist` status=waiting |

## 班次管理 (/admin/schedules)

### 筛选

表单 GET 提交，支持四个筛选条件：

| 参数 | 说明 |
|------|------|
| departure | 出发地模糊匹配 |
| destination | 目的地模糊匹配 |
| status | active / cancelled / 全部 |
| date_from / date_to | 出发时间范围 |

### 操作

每行班次提供三个操作按钮：

| 按钮 | 功能 |
|------|------|
| 编辑 | 跳转到编辑表单 |
| 复制 | 一键复制为新班次（满座位），跳转编辑页 |
| 取消 | 确认后取消班次，关联订单自动取消，候补自动取消 |

### 添加/编辑班次

表单字段：出发地、目的地、出发时间、到达时间、票价、总座位数

验证规则：
- 所有字段必填
- 到达时间必须晚于出发时间
- 票价 > 0
- 总座位数 > 0
- 编辑时：总座位数 ≥ 已售出数量

## 订单管理 (/admin/orders)

### 筛选

| 参数 | 说明 |
|------|------|
| keyword | 用户名 / 乘车人 / 手机号 / 出发地 / 目的地模糊匹配 |
| status | paid / used / cancelled / 全部 |
| date_from / date_to | 下单时间范围 |

### 操作

- **状态切换**：每行下拉框选择新状态，确认后提交
  - paid → used：不变座位
  - paid/used → cancelled：释放座位，触发候补
  - cancelled → paid/used：扣减座位（满座拒绝）
- **导出 CSV**：按当前筛选条件导出，UTF-8 BOM 编码，Excel 可直接打开

### CSV 导出字段

订单号、用户名、乘车人、手机号、出发地、目的地、出发时间、到达时间、座位号、票价、状态、下单时间

## 用户管理 (/admin/users)

### 筛选

| 参数 | 说明 |
|------|------|
| q | 用户名/姓名/手机/邮箱模糊匹配 |
| role | admin / user / 全部 |

### 操作

- **切换管理员**：点击按钮切换用户管理员身份
  - 不能修改自己的管理员状态
  - 切换后即时生效
